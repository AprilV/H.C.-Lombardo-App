# BEST_PRACTICES

## Coding
- Prefer targeted edits over broad refactors.
- Keep behavior stable unless requirement says to change it.
- Add concise comments only where logic is non-obvious.

## Testing And Verification
- Validate changed behavior with direct checks.
- Prefer deterministic checks over manual-only confirmation.
- Capture evidence in final summary.

## Git Hygiene
- Keep commit scope focused.
- Do not include unrelated changes in task commits.
- Preserve user worktree context.

## Documentation Hygiene
- Update docs for behavior/process changes.
- Keep startup runbooks executable and current.

## Communication
- Provide short progress updates during multi-step work.
- Report blockers immediately with concrete next options.
