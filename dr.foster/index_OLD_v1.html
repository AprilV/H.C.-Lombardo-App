October 9, 2025 - Three-Tier Architecture

Production System Components

  • Presentation Layer: React 18.2.0 Frontend (Port 3000)
  • Application Layer: Flask REST API (Port 5000)
  • Data Layer: PostgreSQL 18 Database (Port 5432)

DHCP-Inspired Port Management

Innovation Highlight

Applied DHCP (Dynamic Host Configuration Protocol) networking concepts to solve port conflicts in development.

  • Problem: Frequent "port already in use" errors
  • Solution: Automatic port assignment from managed pool (5000-5010)
  • Testing: 100% pass rate (12/12 unit tests, full integration)

Major Upgrades Completed

1. PostgreSQL Migration

  • Upgraded from SQLite to PostgreSQL 18
  • Enterprise-grade RDBMS with 32 NFL teams
  • Real-time PPG and PA data

2. Live Data Integration

  • Web scraping from TeamRankings.com
  • Automatic 24-hour refresh cycle
  • Real-time Points Per Game (PPG)
  • Real-time Points Allowed (PA)

3. Comprehensive Logging System

  • Daily log files with rotation (10MB max, keeps 5 files)
  • Color-coded output (Info, Warning, Error)
  • Component-based logging
  • Built-in viewers: log_viewer.py, quick_logs.py

Communication Flow

  1. User ‚Üí React: Opens browser at http://localhost:3000
  2. React ‚Üí Flask API: fetch('http://localhost:5000/api/teams')
  3. Flask ‚Üí PostgreSQL: cursor.execute("SELECT * FROM teams")
  4. PostgreSQL ‚Üí Flask: Returns 32 teams with statistics
  5. Flask ‚Üí React: Sends JSON response {"teams": [...]}
  6. React ‚Üí User: Displays teams in beautiful card UI

Key Benefits

  • ‚úÖ Separation of concerns (UI, logic, data)
  • ‚úÖ Scalable (can add more API servers)
  • ‚úÖ Testable (each tier tested independently)
  • ‚úÖ Maintainable (changes in one tier don't affect others)
  • ‚úÖ Secure (database credentials only in API server)
  • ‚úÖ Future-ready (can add mobile apps using same API)
