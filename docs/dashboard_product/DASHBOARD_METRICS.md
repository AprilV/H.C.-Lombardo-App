# Dashboard Metrics Baseline

Date: 2026-04-23

## Purpose

Define measurable indicators for dashboard quality and process maturity.

## Core KPIs

1. Ticket Throughput
- Definition: number of `DSH-` tickets moved to `DONE` per sprint.
- Target: >= 2 per sprint while core capstone scope is active.

2. Lead Time
- Definition: elapsed days from `TODO` to `DONE` for each `DSH-` ticket.
- Target: median <= 14 days.

3. Blocked Time
- Definition: elapsed days in `BLOCKED` status per ticket.
- Target: median <= 3 days.

4. Resolution Completeness
- Definition: percentage of completed sprint subtasks with all required fields.
- Formula: completed subtasks with full metadata / total completed subtasks.
- Target: 100%.

5. Documentation Coverage
- Definition: percentage of dashboard code changes represented in release notes.
- Target: 100%.

## Review Rhythm

1. Weekly check-in: update KPI notes and major deltas.
2. Sprint close: capture final KPI snapshot for the sprint.
